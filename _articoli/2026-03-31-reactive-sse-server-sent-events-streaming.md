## Esigenza Reale
Creare una dashboard che si aggiorna automaticamente man mano che arrivano nuovi dati dal database o da un message broker.

## Analisi Tecnica
Problema: Necessità di aggiornamenti real-time che solitamente richiederebbero polling pesante o l'overhead di un server WebSocket. Perché: Uso il Data-Driven Streaming. Ho scelto SSE con Thymeleaf Reactive per inviare chunk di HTML pronti per il DOM non appena il backend produce un nuovo evento, riducendo la latenza percepita.

## Esempio Implementativo

```thymeleaf
/* Controller WebFlux che espone due endpoint: 1) La pagina HTML iniziale con il container vuoto 2) Lo stream SSE che invia frammenti HTML al client */ @Controller @RequestMapping("/monitoring") public class MonitoringController { @Autowired private MetricsService metricsService; @Autowired private TemplateEngine templateEngine; /* Endpoint per la pagina iniziale: renderizza il layout con il container vuoto. */ @GetMapping public String dashboardPage(Model model) { // Dati iniziali sincroni: gli aggiornamenti arrivano via SSE model.addAttribute("title", "Monitoring Dashboard"); return "monitoring/dashboard"; } /* Endpoint SSE: restituisce frammenti HTML renderizzati lato server. Il client (HTMX o EventSource) riceve HTML pronto da inserire nel DOM. */ @GetMapping(value = "/stream", produces = MediaType.TEXT_EVENT_STREAM_VALUE) public Flux<ServerSentEvent<String>> streamMetrics() { return metricsService.getMetricsStream() // Flux<MetricEvent> da Kafka/Redis/DB .map(metric -> { // Renderizzo il frammento Thymeleaf per ogni evento Context ctx = new Context(); ctx.setVariable("metric", metric); ctx.setVariable("timestamp", LocalDateTime.now()); String html = templateEngine.process( "monitoring/fragments/metric-row", // Frammento specifico ctx ); return ServerSentEvent.<String>builder() .id(metric.getId().toString()) .event("metric-update") // Tipo evento: il client filtra per tipo .data(html) // HTML pre-renderizzato .build(); }) .onErrorResume(e -> { log.error("Errore nello stream metriche", e); return Flux.just(ServerSentEvent.<String>builder() .event("error") .data("<div class='alert alert-danger'>Connessione persa</div>") .build()); }); } /* Per stream con backpressure: limito la frequenza di emissione per non sovraccaricare il client. */ @GetMapping(value = "/logs", produces = MediaType.TEXT_EVENT_STREAM_VALUE) public Flux<ServerSentEvent<String>> streamLogs() { return logService.getLogStream() .sample(Duration.ofMillis(500)) // Massimo 2 aggiornamenti al secondo .window(10) // Raggruppo in batch da 10 log .flatMap(batch -> batch.collectList()) .map(logs -> { Context ctx = new Context(); ctx.setVariable("logs", logs); String html = templateEngine.process("monitoring/fragments/log-batch", ctx); return ServerSentEvent.<String>builder() .event("log-batch") .data(html) .build(); }); } } <!-- Template monitoring/dashboard.html: container che riceve gli aggiornamenti SSE via HTMX. --> <!DOCTYPE html> <html xmlns:th="http://www.thymeleaf.org"> <head> <title th:text="${title}">Dashboard</title> <!-- HTMX gestisce la connessione SSE e l'inserimento degli HTML fragments --> <script src="https://unpkg.com/htmx.org@1.9.10"></script> <script src="https://unpkg.com/htmx.org/dist/ext/sse.js"></script> </head> <body> <h1>Monitoring in Tempo Reale</h1> <!-- Container metriche: HTMX apre la connessione SSE e inserisce gli update --> <div hx-ext="sse" sse-connect="/monitoring/stream" sse-swap="metric-update" <!-- Reagisce all'evento "metric-update" --> hx-swap="afterbegin" <!-- Aggiunge nuove righe in cima --> id="metrics-container"> <p class="text-muted">In attesa di dati...</p> </div> <!-- Container log: stream separato con evento diverso --> <div hx-ext="sse" sse-connect="/monitoring/logs" sse-swap="log-batch" hx-swap="beforeend" hx-target="#log-list"> <ul id="log-list"></ul> </div> </body> </html> <!-- Frammento monitoring/fragments/metric-row.html: renderizzato lato server per ogni evento. --> <!-- <div th:fragment="metric-row" class="metric-card"> --> <!--   <span th:text="${metric.name}">CPU Usage</span> --> <!--   <span th:class="'badge ' + (${metric.value > 80} ? 'bg-danger' : 'bg-success')" --> <!--         th:text="${metric.value + '%'}">0%</span> --> <!--   <small th:text="${#temporals.format(timestamp, 'HH:mm:ss')}">00:00:00</small> --> <!-- </div> -->
```
